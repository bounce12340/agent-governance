# About / 關於這個專案

## English

This project explores a simple idea:
if AI agents are going to make decisions, they should also be governed.

The framework borrows from **separation of powers**:

- **Legislative** defines what success means.
- **Executive** builds the thing.
- **Judiciary** checks whether the thing is actually acceptable.

The extra layer is **Harness Engineering**.
That means every claim must be testable, every output must be verifiable, and every failure should produce useful evidence.

This is not about making AI slower.
It is about making AI accountable.

## 繁體中文

這個專案想回答一個很簡單的問題：
如果 AI Agent 真的要參與決策，那它們也應該被治理。

這套框架借用的是 **三權分立**：

- **立法權** 定義什麼叫成功。
- **行政權** 負責把東西做出來。
- **司法權** 負責檢查成果到底合不合格。

另外再加上一層 **Harness Engineering**。
意思是每個主張都要能驗證、每個輸出都要能審查、每次失敗都要留下有用證據。

這不是為了讓 AI 變慢。
而是為了讓 AI 變得可問責。

## Design goals / 設計目標

- Clear contracts / 清楚契約
- Repeatable verification / 可重複驗證
- Fast feedback / 快速回饋
- Strong audit trail / 完整稽核軌跡
- Bilingual docs / 雙語文件

## Governance improvements / 治理改良

This repo now includes a constitution layer, harness gate, role isolation, and reverse feedback paths.

這個 repo 現在加入了憲法層、harness gate、角色隔離，以及反向回饋通道。

## What this is not / 這不是什麼

- Not free-form agent chat / 不是自由聊天型 Agent
- Not prompt chaos / 不是 prompt 大雜燴
- Not trust me bro AI / 不是「相信我就好」的 AI

## Who it is for / 適合誰

- Builders who want reliable AI workflows
- Teams that need reviewable agent output
- People who like systems with rules, tests, and evidence

- 想要可靠 AI 流程的開發者
- 需要可審查輸出的團隊
- 喜歡有規則、有測試、有證據的系統的人
