# Example: App Launch Law
# 範例：APP 上架法律

## English

### Legislative output
- Target: build a consumer app in 4 to 8 weeks
- Market research: at least 30 valid responses, 5 interviews
- MVP: one core flow, one account system, one persistent storage path
- Red lines: privacy policy, no deceptive UI, no forbidden payment bypass

### Executive output
- Implement the core flow
- Build a working iOS/Android-ready app
- Provide logs, screenshots, and a test checklist

### Judiciary check
- Does the product match the law?
- Does the core flow pass the test harness?
- Are the red lines respected?

### Pass criteria
- All required fields present
- Core flow succeeds without crash
- No policy violations

## 繁體中文

### 立法權輸出
- 目標：在 4 到 8 週內做出一個可上架的消費型 APP
- 市場調查：至少 30 份有效回覆、5 場訪談
- MVP：一條核心流程、一套帳號系統、一個資料持久化路徑
- 紅線：隱私政策、不能誤導 UI、不能繞過支付規範

### 行政權輸出
- 實作核心流程
- 做出可供 iOS / Android 上架的版本
- 提供 log、截圖、測試清單

### 司法權審查
- 產品是否符合法律？
- 核心流程是否通過 harness 測試？
- 是否踩到紅線？

### 通過條件
- 所有必填項都齊全
- 核心流程無 crash 通過
- 沒有違規項目
