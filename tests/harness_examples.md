# Harness Test Examples
# Harness 測試例子

## Example 1, market research validation
## 範例 1，市場調查驗證

- Input: survey results, interview notes, competitor table
- Expected: at least 30 valid responses, 5 interviews, 3 competitors
- Failure: missing counts, duplicated responses, no supporting evidence

- 輸入：問卷結果、訪談筆記、競品表
- 預期：至少 30 份有效回覆、5 場訪談、3 個競品
- 失敗：數量不足、重複回覆、沒有證據

## Example 2, MVP acceptance test
## 範例 2，MVP 驗收測試

- Input: installable app build
- Expected: core flow works end to end
- Failure: crash, blocked flow, missing privacy policy

- 輸入：可安裝的 app build
- 預期：核心流程端到端可運作
- 失敗：閃退、流程卡死、缺少隱私政策

## Example 3, judiciary review
## 範例 3，司法審查

- Input: executive output + evidence bundle
- Expected: every law item has proof
- Failure: unsupported claim, vague result, policy violation

- 輸入：行政權輸出 + 證據包
- 預期：每一條法律都有證據
- 失敗：沒有證據的主張、模糊結果、違規內容
